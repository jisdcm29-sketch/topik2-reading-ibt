TOPIK II 읽기 PBT형 IBT 시뮬레이션 - reading-diagnosis
============================================================

1. 프로그램 목적
------------------------------------------------------------
이 폴더는 TOPIK II 읽기 PBT형 IBT 시뮬레이션의 진단 보고서 화면입니다.

reading-diagnosis는 시험을 실행하지 않습니다.
시험 실행, 채점, 결과 저장은 reading-test에서 담당합니다.

역할 분리:
- reading-test: 시험 실행, 답안 저장, 채점, 결과 생성, localStorage 저장
- reading-diagnosis: 결과 분석, 예상 수준, 약점 진단, 학습 처방, PDF 인쇄 / 저장

이 프로그램은 공식 TOPIK II IBT 프로그램을 복제한 것이 아닙니다.
TOPIK II PBT 읽기 1~50번 문제를 컴퓨터 화면에서 풀 수 있게 만든
PBT형 IBT 시뮬레이션의 진단 보고서입니다.

4. 진단 보고서 실행 주소
------------------------------------------------------------
로컬 서버 실행 후 아래 주소로 접속합니다.

http://localhost:5500/reading-diagnosis/index.html?auto=1

일반적으로 학생은 이 주소를 직접 입력하지 않습니다.
reading-test에서 시험 제출 후 [진단 보고서 보기] 버튼을 누르면 자동으로 연결됩니다.


5. 로컬 실행 방법
------------------------------------------------------------
file:/// 직접 실행은 권장하지 않습니다.
반드시 로컬 서버에서 실행합니다.

PowerShell에서 실행:

cd C:\topik2-separated-system
python -m http.server 5500

시험 실행 주소:

http://localhost:5500/reading-test/index.html

진단 보고서 주소:

http://localhost:5500/reading-diagnosis/index.html?auto=1

캐시 확인용 예:

http://localhost:5500/reading-diagnosis/index.html?auto=1&v=diagnosis-check-1


6. 진단 보고서가 읽는 localStorage 키
------------------------------------------------------------
진단 보고서는 reading-test가 저장한 localStorage 값을 읽습니다.

주요 키:

topik2_latest_reading_result
- 첫 번째 50문항 일반 시험 결과
- 진단 보고서의 기준 결과
- 오답풀이 결과가 이 값을 덮어쓰면 안 됩니다.

topik2_wrong_review_question_numbers
- 오답 다시 풀기 대상 문항 번호
- 틀린 문항과 미응답 문항을 모두 포함합니다.
- 남은 오답이 0개이면 삭제하지 않고 []로 유지합니다.

topik2_wrong_review_source_result
- 오답풀이 연결에 필요한 응시자 정보와 복습 상태
- 오답풀이 모드에서 이름과 전화번호를 자동으로 사용할 때 참고합니다.

9. 예상 수준 표시 원칙
------------------------------------------------------------
진단 보고서의 예상 수준은 읽기 영역 점수만 기준으로 한 참고용입니다.

공식 TOPIK II 급수는 듣기, 쓰기, 읽기 총점 300점 기준으로 결정됩니다.
따라서 이 보고서만으로 공식 급수를 확정할 수 없습니다.

보고서에는 반드시 다음 취지의 안내가 포함되어야 합니다.

- 이 보고서는 TOPIK II 읽기 영역만 기준으로 한 예상 수준입니다.
- 공식 TOPIK II 급수는 듣기·쓰기·읽기 총점 기준으로 결정됩니다.
- 이 결과만으로 공식 급수를 확정할 수 없습니다.


10. 오답 다시 풀기 원칙
------------------------------------------------------------
진단 보고서의 오답 다시 풀기 버튼은 다음 문항을 대상으로 합니다.

1) 틀린 문항
2) 미응답 문항

중요:
미응답 문항도 다시 풀어야 하므로 오답풀이 대상에 포함합니다.

오답 다시 풀기 흐름:
1. 학생이 50문항 시험을 제출합니다.
2. reading-test가 topik2_latest_reading_result를 저장합니다.
3. reading-test가 topik2_wrong_review_question_numbers도 저장합니다.
4. 진단 보고서가 이 값을 읽어 오답 다시 풀기 버튼을 표시합니다.
5. 버튼을 누르면 reading-test/index.html?mode=wrong-review로 이동합니다.
6. 오답풀이에서는 인증을 생략하고 바로 시작합니다.
7. 오답풀이 결과는 topik2_latest_reading_result를 덮어쓰지 않습니다.
8. 오답풀이 후 남은 오답 번호만 topik2_wrong_review_question_numbers에 다시 저장합니다.
9. 진단 보고서로 돌아오면 남은 오답 수가 갱신되어야 합니다.

주의:
남은 오답이 없을 때도 topik2_wrong_review_question_numbers는 []로 저장해야 합니다.
이 키를 삭제하면 진단 보고서가 원래 첫 시험의 오답 목록으로 되돌아갈 수 있습니다.


11. PDF 인쇄 / 저장 원칙
------------------------------------------------------------
진단 보고서 화면에는 학생용으로 다음 버튼만 보이게 합니다.

- PDF로 인쇄 / 저장
- 오답 다시 풀기 또는 남은 오답 안내

JSON 다운로드, TXT 다운로드 같은 개발자용 버튼은 진단 보고서 화면에 보이지 않게 합니다.

PDF 저장은 브라우저 인쇄 기능을 사용합니다.
버튼을 누르면 window.print() 방식으로 인쇄 창이 열리고,
사용자는 프린터 대신 PDF 저장을 선택할 수 있습니다.


12. 오답풀이 결과와 진단 보고서 분리
------------------------------------------------------------
진단 보고서는 항상 첫 번째 50문항 일반 시험 결과를 기준으로 유지합니다.

오답풀이 결과는 다음 용도로만 사용합니다.

- 남은 오답 번호 갱신
- 다시 틀린 문제만 반복 풀이
- 오답풀이 결과 화면 표시

오답풀이 결과가 topik2_latest_reading_result를 덮어쓰면 안 됩니다.
덮어쓰면 진단 보고서가 1문항 또는 일부 문항 기준으로 잘못 표시될 수 있습니다.

15. GitHub Pages 배포 시 주의
------------------------------------------------------------
1. 상대 경로를 유지합니다.
2. reading-test와 reading-diagnosis의 폴더 분리를 유지합니다.
3. localStorage 키 이름을 바꾸지 않습니다.
4. index.html과 reading-diagnosis.js 파일 이름을 바꾸지 않습니다.
5. file:/// 직접 실행으로 확인하지 않습니다.
6. GitHub Pages 배포 후 Ctrl+F5로 새로고침합니다.
7. 캐시 확인을 위해 URL 뒤에 ?v=버전명을 붙입니다.

예:

reading-diagnosis/index.html?auto=1&v=deploy-check-1


16. 수정 시 주의사항
------------------------------------------------------------
1. reading-diagnosis에 시험 실행 코드를 넣지 않습니다.
2. reading-test에 진단 분석 코드를 넣지 않습니다.
3. result.json 필수 구조를 깨뜨리지 않습니다.
4. 오답풀이 결과가 첫 50문항 진단 결과를 덮어쓰지 않게 합니다.
5. 버튼을 삭제하지 말고 필요하면 숨김 처리합니다.
6. HTML과 JS가 같은 버튼을 각각 만들지 않게 합니다.
7. PDF 자동 저장을 과장하지 않습니다. 브라우저 인쇄 기능을 사용합니다.
8. 공식 TOPIK 급수를 확정한다고 표현하지 않습니다.
9. 읽기 영역 기준 예상 수준이라고 안내합니다.
10. 캐시 문제를 코드 오류로 단정하지 않습니다.
