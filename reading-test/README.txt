TOPIK II 읽기 PBT형 IBT 시뮬레이션 - reading-test
============================================================

1. 프로그램 목적
------------------------------------------------------------
이 프로그램은 공식 TOPIK II IBT 프로그램을 복제한 것이 아닙니다.

TOPIK II PBT 읽기 1~50번 문제를 컴퓨터 화면에서 풀 수 있도록 만든
PBT형 IBT 시뮬레이션입니다.

현재 reading-test는 시험 실행만 담당합니다.
진단 분석과 PDF 보고서는 reading-diagnosis에서 담당합니다.

역할 분리:
- reading-test: 시험 실행, 채점, 결과 저장, 진단 보고서 연결
- reading-diagnosis: 결과 분석, 예상 수준, 약점 진단, 학습 처방, PDF 인쇄 / 저장

3. reading-test 주요 파일
------------------------------------------------------------
index.html
- 시험 시작 화면
- 인증 비밀번호 입력
- 응시자 이름 입력
- 전화번호 입력
- 시험지 선택
- 시험 화면
- 결과 요약 화면

reading-test.js
- 인증 처리
- 시험지 불러오기
- 70분 타이머
- 이전 / 다음 / 전체 문제 이동
- 답안 저장
- 제출
- 채점
- topik2-reading-result.json 구조 생성
- localStorage 저장
- 진단 보고서 연결
- 오답 다시 풀기 모드 실행

reading-questions.json
- 기본 시험 실행용 문제 파일
- 현재 103회 TOPIK II 읽기 1~50번 구조 확인용으로 사용 가능

generated-reading-questions.json
- 회차 고정 출제 또는 생성된 시험지를 실행하기 위한 문제 파일
- 현재 103회 고정 출제 실행용으로 사용

question-bank.json
- 문제은행 파일
- 향후 여러 회차 문제를 type, category, diagnostic_area 기준으로 관리

exam-template.json
- 1~50번 위치별 유형 구조를 정의하는 파일
- 향후 랜덤 출제 시 사용

question-generator.js
- 향후 회차 고정 출제 / 랜덤 출제용 문제 생성 로직에 사용

images
- 이미지, 표, 안내문, 그래프 자료 저장 폴더
- 현재 103회 읽기 5~10번 이미지 문항에 사용


4. 현재 기본 시험지
------------------------------------------------------------
현재 기본 시험지는 다음과 같습니다.

시험지:
103회 고정 출제

문항:
TOPIK II 읽기 1~50번

시간:
70분

점수:
100점

기본 선택값:
103회 고정 출제

현재 시험지 선택 화면 상태:
- 103회 고정 출제: 선택 가능
- 103회 구조 샘플 7문항: 표시되지만 선택 불가
- 랜덤 출제 준비 중: 표시되지만 선택 불가

주의:
다른 회차 문제지를 추가하면 exam-list.json 방식 또는 별도 회차 선택 구조로 확장할 예정입니다.
현재는 103회 고정 출제를 안정화한 상태입니다.


5. 현재 정상 확인된 기능
------------------------------------------------------------
1) 인증 비밀번호 입력
2) 응시자 이름 입력
3) 전화번호 입력
4) 시험지 선택
5) 103회 고정 출제 불러오기
6) TOPIK II 읽기 1~50번 시험 실행
7) 70분 타이머
8) 이전 / 다음 이동
9) 전체 문제 이동
10) 마지막 문항 제출
11) 결과 요약 화면
12) 진단 보고서 보기 버튼
13) localStorage를 통한 reading-diagnosis 자동 연결
14) 진단 보고서 PDF 인쇄 / 저장
15) 오답 다시 풀기
16) 미응답 문항도 오답풀이 대상으로 포함
17) 오답풀이 결과가 첫 번째 50문항 진단 보고서를 덮어쓰지 않음
18) 오답풀이 후 남은 오답 수 갱신
19) 다시 틀린 문제만 반복 풀이
20) JSON/TXT 다운로드 버튼은 학생 화면에서 숨김 처리


6. 현재 반영된 주요 문항 유형
------------------------------------------------------------
현재 103회 읽기 문항 기준으로 다음 유형을 반영했습니다.

1~4번:
문법·표현 / 밑줄 표현 문제

5~10번:
이미지·자료 문항

13~15번:
문장 순서형
- 학생 배열 영역
- 선택지 문장 영역
- 클릭 또는 드래그로 배열
- 배열 기준 채점

16~18번:
빈칸 삽입형
- 선택지를 누르면 지문의 빈칸에 반영

19~20번:
공통 지문 빈칸 세트
- 19번 선택 결과가 20번 지문에도 반영

21~24번:
공통 지문 기초 세트

25~27번:
신문 제목 / 함축 의미

28~38번:
중·장문 독해

39~41번:
문장 삽입형
- 주어진 문장 표시
- 선택 위치에 주어진 문장 삽입 표시

42~50번:
고난도 장문 세트

8. localStorage 사용 키
------------------------------------------------------------
현재 사용하는 주요 localStorage 키는 다음과 같습니다.

topik2_latest_reading_result
- 첫 번째 50문항 일반 시험 결과 저장
- reading-diagnosis가 자동으로 읽는 기준 결과

topik2_wrong_review_question_numbers
- 오답 다시 풀기 대상 문항 번호 저장
- 틀린 문항과 미응답 문항을 모두 포함

topik2_wrong_review_source_result
- 오답풀이용 응시자 정보와 복습 상태 저장

중요:
오답풀이 결과는 topik2_latest_reading_result를 덮어쓰지 않습니다.
진단 보고서는 첫 번째 50문항 일반 시험 결과를 기준으로 유지합니다.


9. 오답 다시 풀기 원칙
------------------------------------------------------------
오답 다시 풀기에는 다음 문항이 포함됩니다.

1) 틀린 문항
2) 미응답 문항

오답풀이 흐름:
1. 일반 50문항 시험 제출
2. 진단 보고서 보기
3. 진단 보고서 상단의 오답 다시 풀기 버튼 클릭
4. 인증 없이 오답풀이 시작
5. 오답풀이 중간 종료 가능
6. 남은 오답 문항 수 저장
7. 진단 보고서로 돌아가기
8. 다시 오답 다시 풀기를 누르면 남은 오답만 출제

주의:
남은 오답이 0개일 때도 localStorage 값은 []로 유지해야 합니다.
키를 삭제하면 진단 보고서가 원래 오답 목록으로 되돌아갈 수 있습니다.


10. 진단 보고서 연결
------------------------------------------------------------
진단 보고서 주소:

../reading-diagnosis/index.html?auto=1

reading-test에서 시험 제출 후 진단 보고서 보기 버튼을 누르면
localStorage의 topik2_latest_reading_result를 기준으로 자동 연결됩니다.

진단 보고서에서 확인할 수 있는 내용:
- 응시자
- 읽기 점수
- 정답 수
- 미응답 수
- 예상 수준
- 시험 정보
- 문항 구간별 분석
- 유형별 분석
- 진단 영역별 분석
- 강점 영역
- 약점 영역
- 오답 문항
- 미응답 문항
- 학습 처방
- 2주 학습 계획
- PDF로 인쇄 / 저장


11. 로컬 실행 방법
------------------------------------------------------------
file:/// 직접 실행은 권장하지 않습니다.
반드시 로컬 서버로 실행합니다.

PowerShell에서 실행:

cd C:\topik2-separated-system
python -m http.server 5500

브라우저 주소:

http://localhost:5500/reading-test/index.html

진단 보고서 주소:

http://localhost:5500/reading-diagnosis/index.html?auto=1

캐시 확인용 예:

http://localhost:5500/reading-test/index.html?v=final-check-1

13. GitHub Pages 배포 시 주의
------------------------------------------------------------
1. 상대 경로를 유지합니다.
2. images 폴더 경로가 깨지지 않게 합니다.
3. JS, JSON 파일 이름을 바꾸지 않습니다.
4. file:/// 직접 실행으로 확인하지 않습니다.
5. GitHub Pages 배포 후 Ctrl+F5로 새로고침합니다.
6. 캐시 문제 확인을 위해 URL 뒤에 ?v=버전명을 붙입니다.

예:

reading-test/index.html?v=deploy-check-1
